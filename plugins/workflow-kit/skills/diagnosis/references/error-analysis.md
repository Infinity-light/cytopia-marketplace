# 错误分析方法

> 供Subagent分析错误时参考

## 错误分类

### 编译/构建错误
- 语法错误
- 类型错误
- 模块未找到
- 配置错误

### 运行时错误
- 空指针/undefined
- 类型不匹配
- 资源不存在
- 权限不足

### 逻辑错误
- 结果不符合预期
- 边界条件处理不当
- 状态不一致

---

## 错误信息解读

### 堆栈追踪
1. 从上往下读
2. 找到自己代码的第一行
3. 那通常是问题发生点

### 常见错误模式

| 错误信息 | 可能原因 |
|---------|---------|
| Cannot read property 'x' of undefined | 对象为undefined |
| x is not a function | 变量不是函数类型 |
| Module not found | 导入路径错误或未安装 |
| ECONNREFUSED | 服务未启动或端口错误 |
| 401 Unauthorized | 认证失败 |
| 403 Forbidden | 权限不足 |
| 404 Not Found | 资源不存在 |
| 500 Internal Server Error | 服务端异常 |

---

## 分析步骤

### 1. 读懂错误
- 错误类型是什么
- 错误消息说了什么
- 发生在哪个文件哪一行

### 2. 追溯来源
- 这个值从哪里来
- 什么条件导致这个状态
- 数据流是怎样的

### 3. 检查假设
- 我假设这个值是什么
- 实际是什么
- 为什么不一致

### 4. 验证修复
- 修复后错误消失
- 功能正常工作
- 无新问题引入
