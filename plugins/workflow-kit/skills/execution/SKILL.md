## 常见借口（都是错的）

| 你可能会想 | 为什么是错的 |
|-----------|------------|
| "这个任务太简单，我直接做了" | 主Agent 直接做 = 跳过审查。再简单也要走子代理 |
| "就改一行代码，不需要子代理" | 一行也是代码。主Agent 碰代码 = 违规。没有例外 |
| "规格审查太慢，先跳过" | 没审查的代码是薛定谔的代码——你不知道它对不对 |
| "子代理上下文够了，不需要传文件路径" | 子代理没有你的上下文。必须指定要读取的文件路径 |
| "不需要扫描进度，我记得哪些文件做完了" | 你的记忆不可靠。扫描脚本的结果才是硬事实 |

## 核心职责

Execution 是契约驱动的实现层。文件头部的 YAML 契约是实现规格，测试骨架是验收标准，CLAUDE.md 是全局上下文。

## 契约驱动开发（Contract-Driven Development）

### 0. 环境准备

运行进度扫描脚本，读取 `.claude/progress.md`：
```bash
python -X utf8 .claude/scripts/scan_contracts.py <项目根目录>
```
- 确认当前项目状态
- 识别可执行文件（PENDING 且依赖全部 VERIFIED）

### 1. 确定下一个任务

从 `.claude/progress.md` 的"下一步可执行"部分：
- 找到状态为 PENDING 且依赖全部 VERIFIED 的文件
- 按依赖顺序选择第一个可执行文件
- 如果没有 PENDING 但有 IMPLEMENTED → 需要审查验证
- 如果全部 VERIFIED → 所有任务完成

### 2. 匹配 Workflow

根据任务内容，到 `workflows/` 目录下探索，找到最匹配的 workflow reference：
- 用 Glob 扫描 `workflows/` 下的文件列表
- 根据文件名和任务类型判断匹配度
- 读取匹配的 workflow，按其中的流程指引执行

如果没有匹配的 workflow，按通用逻辑执行。

### 3. TDD RED 阶段

启动子代理 A（实现测试断言）：

prompt 模板：
```
实现 [测试文件路径] 中所有 TODO 测试用例的断言。

开始前先读取以下文件：
1. [目标文件路径]（头部 --- 契约块是测试依据）
2. [测试文件路径]（将 TODO 替换为具体断言）
3. 项目 CLAUDE.md（全局上下文）

规则：
- 根据契约中每个函数的行为描述编写断言
- 写完后运行测试，确认测试失败（RED）
- 如果测试意外通过，说明断言写得不对，需要修正
```

### 4. TDD GREEN 阶段

启动子代理 B（实现代码）：

prompt 模板：
```
实现 [目标文件路径] 中契约定义的所有函数。

开始前先读取以下文件：
1. [目标文件路径]（头部 --- 契约块是实现规格，不要修改 --- 契约块内容）
2. [测试文件路径]（必须让所有测试通过）
3. 项目 CLAUDE.md（全局上下文和开发规范）

规则：
- 在 --- 契约块下方写实现代码
- 不要修改 --- 契约块的内容（除 status 字段外）
- 实现完成后运行测试，确认全部通过（GREEN）
- 通过后将契约中 status 改为 IMPLEMENTED
```

### 5. 规格审查

启动审查子代理（独立于实现子代理）：

prompt 模板：
```
审查 [目标文件路径] 的实现是否完全符合头部契约。

读取文件头部的 --- 契约块，逐条检查：
- 每个 functions 中列出的函数是否都已实现
- 实现是否符合行为描述（包括边界条件和错误处理）
- 依赖关系是否正确引用
- 运行测试确认全部通过

审查通过 → 将契约中 status 改为 VERIFIED
审查不通过 → 列出不符合项，不修改 status
```

审查不通过 → 启动新子代理修复 → 重新审查

### 6. 更新进度

审查通过后：
- 运行进度扫描脚本，更新 `.claude/progress.md`
- 回到步骤 1 确定下一个文件

### 7. 涌现处理

如果实现过程中发现契约有问题（遗漏边界条件、接口设计不合理）：
- 子代理暂停实现，报告问题
- 主Agent 评估：
  - 小问题（不影响其他文件）→ 启动子代理修改当前文件契约，继续实现
  - 大问题（影响多个文件的接口）→ 暂停 Execution，用 AskUserQuestion 告知用户，必要时回到 Planning 修正

### 8. 全部完成

所有文件 VERIFIED 后：
- 运行最终进度扫描，确认 `.claude/progress.md` 显示 100%
- 简要告知用户结果

## 执行中遇到不确定情况

1. 暂停执行
2. 进行必要的探索（查代码、查文档、查 references/）
3. 确认后继续

## 阶段结束

所有任务完成后，用 AskUserQuestion 向用户确认下一步：

选项必须包含：
- "进入 Deploy"（推荐）
- "需要补充任务"
- "暂停"
