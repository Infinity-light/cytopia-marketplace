# 前端设计验证清单

验证子代理使用此清单，逐项检查产出子代理的前端设计方案。

## 每个页面必须包含

- [ ] 布局描述（文字或ASCII线框，说明各区域位置关系）
- [ ] 组件层级树（父子关系，不是平铺列表）
- [ ] 核心交互流程（用户操作 → 系统响应 → 页面变化/跳转）
- [ ] 响应式说明（桌面端和移动端的布局差异，至少一句话）

## 全局必须包含

- [ ] 页面间导航关系（从哪到哪，怎么跳转）
- [ ] 认证流程的UI表现（未登录时看到什么、怎么跳转到登录页）
- [ ] 全局组件说明（导航栏、侧边栏等跨页面复用的组件）

## 禁止项

- [ ] 不得出现"待定"、"后续补充"等占位描述
- [ ] 不得遗漏空状态设计（无数据时用户看到什么）
