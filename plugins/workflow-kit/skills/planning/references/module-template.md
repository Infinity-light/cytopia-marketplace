# 模块设计文档强制模板

产出子代理必须严格按此模板结构编写每个模块的设计文档。每个章节都是必填项，不得省略。

---

```markdown
# {模块名}

## 1. 职责与边界

**做什么**：（一段话描述核心职责）

**不做什么**：（明确列出不属于本模块的功能）

**依赖的上游模块**：（列出模块名，无依赖则写"无"）

## 2. 后端

### 2.1 数据模型

涉及的表及关键字段。标注与其他模块共享的表。

| 表名 | 关键字段 | 说明 | 共享 |
|------|---------|------|------|
| xxx  | id, ... | ...  | 是/否 |

### 2.2 API 规格

每个接口按以下格式：

#### {接口名称}

- **方法 + 路径**：`POST /api/xxx`
- **认证**：需要 / 不需要
- **请求体**：
```json
{ "field": "type — 说明" }
```
- **成功响应 200**：
```json
{ "field": "type — 说明" }
```
- **错误响应**：
  - 401：`{ "message": "Unauthorized" }`
  - 400：`{ "message": "具体错误描述" }`

### 2.3 业务逻辑

核心处理流程（伪代码级别）：

```
1. 验证输入
2. 查询数据
3. 处理逻辑
4. 存储结果
5. 返回响应
```

## 3. 前端

### 3.1 页面与路由

| 路由路径 | 页面名称 | 说明 |
|---------|---------|------|
| /xxx    | XxxPage | ...  |

### 3.2 组件树

```
PageComponent
├── HeaderSection
│   └── ...
├── MainContent
│   ├── ComponentA (props: ...)
│   └── ComponentB (props: ...)
└── FooterActions
```

### 3.3 关键组件 Props

| 组件名 | Prop | 类型 | 必填 | 说明 |
|--------|------|------|------|------|
| xxx    | xxx  | xxx  | 是/否 | ... |

### 3.4 页面状态

| 状态 | 显示内容 |
|------|---------|
| loading | （骨架屏 / Spinner / 具体描述） |
| empty | （引导文案，如"暂无数据，去xxx"） |
| error | （错误提示 + 重试按钮） |
| normal | （正常数据展示） |

### 3.5 数据获取

| 页面 | API | 方式 | 环境变量 |
|------|-----|------|---------|
| xxx  | GET /api/xxx | CSR/SSR/SWR | NEXT_PUBLIC_API_URL |

## 4. 模块间接口

**暴露给其他模块**：
- （函数签名 / 事件名 / API路径）

**依赖其他模块**：
- （具体到函数名或API路径）

## 5. 外部资源依赖

本模块需要的外部资源（不由本项目代码提供的东西）。无则写"无"。

| 资源类型 | 具体需求 | 用途 | 是否必须 |
|---------|---------|------|---------|
| API服务 | 如：OpenAI API / TTS服务 | 用于xxx | 是/否 |
| 数据源 | 如：雅思真题题库 / 音频文件 | 用于xxx | 是/否 |
| 第三方SDK | 如：支付SDK / OSS SDK | 用于xxx | 是/否 |
| 其他 | 如：字体文件 / 图标库 | 用于xxx | 是/否 |

## 6. 任务序列

按执行顺序排列。Execution 子代理读取本节逐个执行。

### 任务 6.1: {任务名}
- **文件**：`path/to/file.ts`
- **变更**：（具体做什么，伪代码级别）
- **验证**：`命令`
- **预期**：（命令输出应该是什么）
- **前置**：无 / 任务 6.x

### 任务 6.2: {任务名}
- **文件**：`path/to/file.ts`
- **变更**：...
- **验证**：...
- **预期**：...
- **前置**：任务 6.1
