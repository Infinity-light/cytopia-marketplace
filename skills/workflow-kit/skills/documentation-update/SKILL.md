# 文档更新阶段

## 🎯 核心原则

- **文档与代码必须保持一致**
- **使用项目中的 `CLAUDE.md`**，不再使用"项目架构.md"
- **及时更新**，不要等到项目结束
- **描述"为什么"**，而不仅仅是"是什么"

## 📋 文档更新流程

### 第一步：定位需要更新的文档

- 查找项目根目录或 `.claude/` 目录下的 `CLAUDE.md` 文件
- 如果不存在，创建新的 `CLAUDE.md` 文件
- 使用 Glob 和 Grep 工具查找相关文档

### 第二步：分析代码变更

- 回顾本次开发的所有代码变更
- 识别需要在文档中体现的内容：
  - 新增的模块和功能
  - 修改的架构设计
  - 重要的技术决策
  - 配置项的变更
  - API接口的变更

### 第二步半：扫描文件契约

运行进度扫描脚本获取项目当前状态：
```bash
python -X utf8 .claude/scripts/scan_contracts.py <项目根目录>
```

从所有 VERIFIED 状态的文件契约中提取：
- 新增的模块和职责（从 role 字段）
- 新增的函数和接口（从 functions 字段）
- 模块间依赖关系（从 depends 字段）

这些信息应同步到 CLAUDE.md 的对应章节：
- 新模块 → "核心模块说明"章节
- 新接口 → "API文档"章节
- 新依赖关系 → "架构设计"章节

### 第三步：更新文档内容

- 使用 Edit 工具更新现有文档
- 确保描述准确、完整
- 添加必要的示例代码
- 说明设计决策的原因

### 第四步：验证文档质量

- 使用文档质量检查清单验证
- 确保所有引用的代码、文件、配置都存在
- 确保描述与实际实现一致

## 🔧 文档修复工作流

当发现文档问题时，按以下流程修复：

### 1. 定位所有相关文档

- 使用 Grep 搜索相关关键词
- 找出所有提到该内容的文档
- 列出需要修改的文档清单

### 2. 追溯根本原因

- 为什么文档会出现错误？
- 是代码变更后未同步？
- 还是最初就理解错误？
- 是否有其他类似问题？

### 3. 制定修复计划

- 确定正确的描述
- 列出所有需要修改的位置
- 确定修改的优先级

### 4. 系统性修复

- 按计划逐一修复
- 确保所有相关文档都已更新
- 避免遗漏任何引用

### 5. 验证效果

- 使用文档质量检查清单验证
- 确认问题已完全解决
- 检查是否引入新问题

## 📝 CLAUDE.md 文档结构建议

项目中的 `CLAUDE.md` 应包含以下内容：

### 1. 项目概述

- 项目名称和简介
- 主要功能和目标
- 技术栈

### 2. 架构设计

- 整体架构图
- 模块划分
- 数据流向
- 关键设计决策及原因

### 3. 开发规范

- 代码风格
- 命名规范
- 目录结构
- 提交规范

### 4. 核心模块说明

- 每个核心模块的职责
- 关键函数和类
- 配置项说明
- 使用示例

### 5. API文档

- 接口列表
- 请求/响应格式
- 错误码说明

### 6. 部署和运维

- 环境要求
- 部署步骤
- 配置说明
- 常见问题

## 📖 README 更新规范

README.md 是项目的门面，应该简洁明了，只包含最核心的信息。

### README 应该包含的内容

#### 1. 项目简介

- **项目名称**：清晰的项目名称
- **一句话描述**：用一句话说明项目是做什么的
- **主要功能列表**：3-5个核心功能点

示例：
```markdown
# 项目名称

简短的一句话描述项目的核心功能和目标。

## 主要功能

- 功能点1：简要说明
- 功能点2：简要说明
- 功能点3：简要说明
```

#### 2. 启动方式

使用统一的格式，每一步都用 `# 序号. 步骤说明` 的方式分隔。

**标准模板**（使用项目名称作为conda环境名）：

```markdown
## 快速开始

# 1. 创建conda环境（Python 3.13+）
conda create -n <project-name> python=3.13

# 2. 激活环境
conda activate <project-name>

# 3. 安装依赖
pip install -r requirements.txt

# 4. 运行项目
python main.py
```

**如果需要重建环境**：

```markdown
# 1. 退出当前环境
conda deactivate

# 2. 删除旧环境
conda remove -n <project-name> --all

# 3. 重新创建（带Python 3.13）
conda create -n <project-name> python=3.13

# 4. 激活环境
conda activate <project-name>

# 5. 安装依赖
pip install -r requirements.txt
```

### README 不应该包含的内容

- ❌ 详细的架构设计（应该在 CLAUDE.md 中）
- ❌ 完整的API文档（应该在 CLAUDE.md 中）
- ❌ 开发规范和代码风格（应该在 CLAUDE.md 中）
- ❌ 详细的模块说明（应该在 CLAUDE.md 中）
- ❌ 过多的配置细节（只保留最基本的启动配置）
