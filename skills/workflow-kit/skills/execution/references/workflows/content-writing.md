# Content Writing Workflow

适用于：推文、文案、文章、营销内容、社交媒体帖子等内容创作类任务。

## 上下文探索

执行前，主动探索项目中是否存在以下资源：
- 品牌调性指南、内容规范、风格手册
- 目标平台的格式要求和字数限制
- 历史内容样本（用于学习风格和调性）
- `practices/` 下的写作规范或内容标准
- `practices/` 下的写作规范或内容标准

如果目标平台有对应的外部 Skill（如特定社交平台 Skill），优先加载使用。

## 约束收集

在动笔前，确认以下约束（未明确的主动向用户确认）：
- **平台**：发布在哪里？有什么格式/字数限制？
- **受众**：写给谁看？他们关心什么？
- **调性**：正式/轻松/专业/幽默？
- **目标**：这篇内容要达成什么效果？
- **素材**：用户提供了哪些原始素材或关键信息？

## Subagent 调度

将内容创作任务分配给 Subagent 时，Task prompt 中必须包含：
- 完整的约束信息（平台、受众、调性、格式要求）
- 探索到的规范文件路径（让 Subagent 读取）
- 用户提供的原始素材
- 明确的输出格式要求

## 交付验证

- 内容是否符合平台格式要求（字数、排版、标签）
- 调性是否与品牌/用户要求一致
- 核心信息是否准确传达
- 交付给用户审阅，根据反馈迭代
