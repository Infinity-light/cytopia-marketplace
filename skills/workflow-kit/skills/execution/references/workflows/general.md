# General Workflow

适用于：没有专门 workflow 匹配的任何类型任务的兜底流程。

## 上下文探索

执行前，主动探索：
- 目标文件是否有头部 `---` 契约块（如有，这是最高优先级的实现规格）
- `practices/` 下是否有适用于当前任务的规范
- 项目中是否有 CLAUDE.md（全局上下文和开发规范）
- 项目中是否有相关的历史实现可参考
- 是否有对应的外部 Skill 可加载（优先使用专业 Skill）

## 执行原则

- 如果文件有契约块，严格按契约实现，不要自由发挥
- 每步完成后验证结果，确认符合预期再继续
- 遇到不确定的决策点，暂停向用户确认
- 优先复用已有资源，避免重复造轮子

## Subagent 调度

分配任务给 Subagent 时，Task prompt 中必须包含：
- 明确的任务目标和验收标准
- 需要读取的文件路径列表（契约文件、CLAUDE.md、相关 practices）
- 任务约束条件和边界
- 期望的输出格式

模板：
```
执行 [具体任务描述]。

开始前先读取以下文件：
1. [目标文件路径]（如有 --- 契约块，这是你的实现规格）
2. 项目 CLAUDE.md（全局上下文）
3. [相关 reference 文件路径]

规则：
- 不要修改 --- 契约块的内容（除 status 字段外）
- 遵循 CLAUDE.md 中的开发规范
```

## 交付验证

- 如果文件有契约块，对照契约逐条检查实现是否符合
- 输出是否满足用户的原始需求
- 是否符合探索到的规范要求
- 交付给用户确认，根据反馈迭代
