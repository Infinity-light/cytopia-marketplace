# 架构总览文档验证清单

验证子代理使用此清单，逐项检查架构总览文档。

## 项目概述

- [ ] 项目名称和一句话简介
- [ ] 目标用户描述
- [ ] 技术栈表格（前端、后端、数据库、ORM、部署等）
- [ ] 技术栈中没有"计划使用但实际不用"的技术

## 目录结构

- [ ] 完整的目录树（到二级子目录）
- [ ] 每个目录有一句话说明用途

## 环境变量

- [ ] 列出所有环境变量（变量名 + 用途 + 示例值）
- [ ] 区分了开发环境和生产环境的值
- [ ] 前端环境变量有 `NEXT_PUBLIC_` 或对应框架的前缀标识
- [ ] 没有硬编码的 localhost 或具体域名出现在非环境变量的地方

## 前后端通信约定

- [ ] API base URL 的配置方式（前端从哪个环境变量读取）
- [ ] 统一错误响应格式（JSON结构示例）
- [ ] 认证 token 传递方式（Header名称、格式）
- [ ] 认证流程（登录→获取token→存储→请求携带→过期刷新）

## 数据库设计

- [ ] 使用 Prisma schema 格式（或等效的关系型描述）
- [ ] 表间关系明确（外键、级联规则）
- [ ] 枚举值完整列出
- [ ] 有索引策略（高频查询字段）

## 模块列表

- [ ] 每个模块有名称和一句话职责
- [ ] 模块间依赖关系明确
- [ ] 开发顺序合理（被依赖的模块排在前面）
- [ ] 没有循环依赖
