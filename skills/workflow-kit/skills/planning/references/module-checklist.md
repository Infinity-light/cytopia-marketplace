# 模块文档验证清单

验证子代理使用此清单，对照 `module-template.md` 的强制结构，逐项检查产出子代理的模块文档。

## 结构完整性

- [ ] 包含全部6个章节（职责与边界、后端、前端、模块间接口、外部资源依赖、任务序列）
- [ ] 没有"待定"、"后续补充"、"TODO"等占位内容

## 第1节：职责与边界

- [ ] "做什么"有明确描述
- [ ] "不做什么"有明确描述（不是空的）
- [ ] 上游依赖列出了具体模块名

## 第2节：后端

- [ ] 数据模型列出了涉及的表和关键字段
- [ ] 共享表有标注
- [ ] 每个API有完整的：方法+路径、认证要求、请求体JSON、成功响应JSON、至少一个错误响应
- [ ] API的JSON示例中每个字段有类型说明
- [ ] 业务逻辑有伪代码级别的步骤描述（不是一句话概括）
- [ ] 没有硬编码的 localhost 或具体域名

## 第3节：前端

- [ ] 页面路由表完整
- [ ] 组件树有层级关系（不是平铺列表）
- [ ] 关键组件的Props有类型和必填标注
- [ ] 四种页面状态都有具体描述（loading/empty/error/normal）
- [ ] 数据获取方式明确（CSR/SSR/SWR）
- [ ] 数据获取使用环境变量拼接URL，不是硬编码

## 第4节：模块间接口

- [ ] 暴露的接口有具体签名或路径
- [ ] 依赖的接口能在对应上游模块文档中找到

## 第5节：外部资源依赖

- [ ] 明确列出了外部资源（或写"无"）
- [ ] 每个资源有类型、具体需求、用途、是否必须
- [ ] 没有遗漏（第2、3节中用到的外部服务都在此列出）

## 第6节：任务序列

- [ ] 每个任务有文件路径
- [ ] 每个任务有具体变更描述（伪代码级别，不是"实现xxx功能"）
- [ ] 每个任务有验证命令
- [ ] 每个任务有预期输出
- [ ] 任务间的前置关系合理
- [ ] 任务序列覆盖了第2、3节描述的所有功能点（没有遗漏）
